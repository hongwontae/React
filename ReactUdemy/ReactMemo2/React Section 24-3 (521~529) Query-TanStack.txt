1. refetchType => inValidateQueries의 인자 객체의 프로퍼티
=> Invalid는 기본동작이 기존의 쿼리키에 해당하는 쿼리함수를 무효화하고 다시 실행시켜서 데이터를 가져오는 방식이다.
=> refetchType은 무효화만 시킨다.


2. useParams => React-router-dom의 활성화된 라우트의 동적 세그먼트를 가져올 수 있는 훅이다.


3. 낙관적 업데이트
프론트적으로 업데이트를 먼저하고 백엔드의 결과가 ture면 그대로 두고, 실패하면 에러로 변환

onMutate => mutate 함수가 호출하는 즉시 실행한다. => mutate 프로세스가 완료되기 전에 + 응답을 받기 전에 실행된다.

queryClient => 리액트 쿼리, 캐시 데이터와 상호작용하는 객체

queryClient.	setQueryData();
=> 이미 저장된 데이터의 응답을 기다리지 않고 수정한다. (formdata)
=> 백엔드와 통신하지 않고 요청을 보내는 데이터를 수신해서 업데이트 해버린다.

queryClient.cancelQueries() => 특정 키의 모든 활성 쿼리를 취소한다.
