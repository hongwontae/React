1. 부모 요소가 명시적으로 설정되어야 자식 요소의 height %가 동작한다.


2. 전체 높이를 가지기 위해서는 html, body, #root(index.html의 dom 진입점의 id)가 높이를 100%로 가져야한다.


3. vh/vw : 웹 브라우저에서 웹 페이지가 표시되는 영역이다./ 디바이스에 따라 동적으로 설정된다.
1vh => 뷰포트 높이 1%


4. vmax(최대 뷰포트) 단위이다. : 브라우저 창의 가로와 세로 중 더 큰 쪽의 값을 선택하여 값을 계산한다.
height : 50vmax이고 브라우저의 너비가 1000px, 높이가 700px 일 떄 너비와 높이 중 더 높은 것을 선택한다.
이 떄 너비가 더 높기 떄문에 이를 바탕으로 50% 계산을 실행한다. => height은 500px이 된다.


5. vmin(최소 뷰포트) 단위이다. : 브라우저 창의 가로와 세로 중 더 작은 쪽의 값을 선택하여 값을 계산한다.
width : 50vmin 일 떄 브라우저의 높이가 100px 넓이가 50px일 떄 넓이가 더 작기 때문에 이를 바탕으로 25px이 선택된다.


*** 4,5는 기기의 실질적인 브라우저의 창의 크기와 높이를 대상으로 값을 선택하여 계산하는 것이다.


6. tailwind CSS height
6-1. Height Classes
h-0: 높이 0.
h-full: 부모 요소의 전체 높이.
h-screen: 뷰포트의 100% 높이.
h-auto: 콘텐츠에 따라 자동으로 높이 조정.
h-px, h-1, h-2, ...: 특정 픽셀 높이.

6-2. Min Height Classes:
1. min-h-0: 최소 높이 0.
2. min-h-full: 부모 요소의 전체 최소 높이.
3. min-h-screen => 요소의 최소 높이를 뷰포트의 전체 높이로 설정한다.

6-3. Max Height Classes:
max-h-0: 최대 높이 0.
max-h-full: 부모 요소의 전체 최대 높이.
max-h-screen => 요소의 최대 높이를 뷰포트의 전체 높이로 설정한다.


7. min-width
1. 요소의 최소 가로 크키를 설정한다.
2. 요소가 해당 값보다 작아지 않도록 보장한다.

8. max-width
1. 요소의 최대 가로 크기를 설정한다.
2. 요소가 이 크기보다 커지지 않도록 보장한다.

9. min-height/ max-height
=> 위와 같다.
