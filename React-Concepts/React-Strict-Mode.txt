1. 엄격모드란?
=> react에서 StrictMode라는 내장 컴포넌트가 존재한다.
=> 해당 컴포넌트로 덮으면 Strict-Mode가 실행된다.

=> 모든 컴포넌트 함수를 두 번씩 실행한다.
=> 개발 단계에서만 그렇게 한다.

2. 엄격모드 검사 항목
2-1. 잘못 사용한 생명주기 메서드
=> componentWillMount 같은 지원 종료 메서드(deprecated) 경고

2-2. 레거시 문자열 ref 사용 여부

2-3. findDOMNode 사용 여부

2-4. 예상치 못한 부작용(side effect)
=> 렌더링 과정 (변화를 계산하는 과정, 이전값과 비교한다.)/ 변경을 반영한다. => 커밋
=> Strict 모드는 렌더링 단계의 메서드에 부작용이 있는지를 검사한다.
=> 이 렌더링 과정을 두 번 호출한다.
constructor, render, shouldComponentpdate, getDerivedStateFromProps
함수 컴포넌트 바디
State updater
useStaet, useMemo, useReducer
=> 이중으로 호출한다.

2-5. 레거시 컨택스트 사용 여부
=> Context API를 잘못 사용한 경우를 말하는 것 같다.
