1. Baching이란?
=> 여러 상태 업데이트나 이벤트 처리를 일괄적으로 처리하여 성능을 최적화하는 메커니즘을 의미한다.
=> 모든 상태 변경이나 프로퍼티 변경마다 바로 렌더링하지 않고, 내부적으로 일정 시간동안 상태 변화를 모아둔 후 처리 => 배칭 처리

2. React 배칭 처리 적용 사항
2-1. 함수 내에서 setFunction을 여러 개 호출해도 배칭처리한다.
=> 리액트 18 이전에는 이벤트 핸들러 내에서만 배칭처리되었다./ React 18 이후 이벤트 핸들러 + 일반 함수 + 비동기 작업에서도 배칭처리된다.

2-2. 예를 들어 리액트 18 이전에 fetch내부에서 then 블럭에 state가 1이라면 setCount(count+1)을 두 번 호출하면 값은 3이된다.
=> No 배칭처리
=> 그러나 18 이후에는 배칭처리되어서 값은 2가된다.


3. 리액트의 상태 업데이트 함수는 즉시 새로운 상태를 컴포넌트에 반영하지 않는다.
=> 다음 렌더 사이클에서 적용된다. => 배칭 처리를 위해서 묶어서 처리하기 위함
=> 상태 업데이트를 예약하고 React가 다음 렌더 사이클에서 이를 모조리 처리한다.


4. 함수형 업데이트가 최신값과 배칭처리를 동시에 할 수 있는 이유
4-1. 상태 업데이트를 예약할 떄 이전 상태 값을 인수로 받는다.
