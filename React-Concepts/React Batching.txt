1. 리액트 Batching이란?
정의 : 리액트의 상태값을 일정한 주기로 처리하는 작업을 의미한다.

1-1. 상태값이 변경되었다고 즉시 컴포넌트를 재호출하는 것이 아닌 일정한 규칙, 주기에 의해 재호출을 한다.

1-2. 기존 리액트 17버전까지는 이벤트에 연결된 함수 본문 내에서 직접적으로 변경된 상태값만 배칭의 범위에 있었다.
=> 프로미스와 타이머 등은 포함되지 않았다.
=> 18 버전으로 넘어가면서 프로미스와 타이머 등도 배칭의 범위안에 들어가게 되었다.

1-3. 이벤트, 프로미스, 타이머, 상태값 변경, Native Event를 자동으로 Baching 처리한다. 
=> 이벤트 핸들러 내에서만 Baching 처리하는게 아니다.
=> 이를 자동 배칭이라고 표현한다.
