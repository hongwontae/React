1. 렌더링(Rendering)
정의: 렌더링은 React가 JSX나 컴포넌트로부터 가상 DOM을 생성하는 과정이다.
이 과정에서 React는 컴포넌트의 상태(state)와 속성(props)을 바탕으로 UI를 어떻게 표시할지 결정한다.
이를 "버츄얼 렌더링"이라고 할 수 있습니다.

비교: 렌더링은 실제 DOM에 아직 반영되지 않았기 때문에, 사용자는 이 단계에서의 변화를 볼 수 없다.
 이는 마치 화면에 그려질 그림을 스케치하는 단계와 비슷합니다.


2. 커밋(Commit)
정의: 커밋 단계는 React가 렌더링 과정에서 생성한 가상 DOM의 변경 사항을 실제 DOM에 반영하는 과정이다.
 이 단계에서 변경 사항이 실제로 적용되어 사용자가 UI의 변화를 볼 수 있게 된다.
비교: 커밋은 버츄얼 렌더링된 결과를 실제 화면에 그리는 단계이다.
 가상 DOM과 실제 DOM의 차이를 비교하고, 필요한 최소한의 DOM 업데이트를 수행하여 성능을 최적화한다.


3. 마운트(Mounting)
정의: 마운트는 컴포넌트가 처음으로 DOM에 삽입되는 과정을 의미한다.. 이는 컴포넌트가 처음으로 화면에 그려지는 순간이다.
비교: 마운트는 컴포넌트의 생명주기에서 초기 렌더링 단계에 해당합니다. 컴포넌트가 처음으로 사용자에게 보여지는 순간이며,
 이 때 useEffect의 코드가 실행됩니다(의존성 배열이 없거나 지정된 경우).
