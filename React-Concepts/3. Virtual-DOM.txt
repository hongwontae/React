1 Virtual DOM
1-1. 실제 DOM의 가벼운 복사본이다.

1-2. 작동원리
1. 가상 DOM 트리 생성 : 리액트 Application이 렌더링될 떄, 가상 DOM 트리가 생성된다. 실제 DOM 구조를 반영하지만 메모리에서만 존재한다.
2. 식별 능력 :  State가 변경되면 새로운 가상 DOM을 생성한다. 이전 V-DOM과 새로운 V-DOM를 비교하여 변경된 부분을 식별한다.
3. 업데이트 최적화 : 변경된 부분만 실제 DOM에 반영한다. 렌더링 성능 향상된다.

1-3. 장점
1. 성능 개선 : 실제 DOM 조작은 비용이 많이 드는 작업이다. 가상 DOM을 사용하면 DOM 조작을 최소화 가능
2. 코드 추상화 : 가상 DOM을 사용해서 선언적이고 직관적으로 UI를 설계할 수 있다.
(그냥 DOM을 조작하면 사용하면 document.getByElement 같은 것을 사용해서 난잡하게 변경해야 한다.)
