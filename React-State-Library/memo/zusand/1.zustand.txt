1. zustand란?
=> 간단하고 직관적인 API를 제공하며, 복잡한 설정 없이 애플리케이션의 상태를 효율적으로 관리하는 라이브러리입니다.


2. zustand 장점
2-1. 간단하고 직관적인 사용법
2-2. 자동 리렌더링 최적화
=> 상태 변경 시 해당 상태만을 구독한 컴포넌트만 리렌더링됩니다.
2-3. 미들웨어 지원
=> 로컬 스토리지에 상태를 저장하는 persist 미들웨어나, Redux DevTools와 연동할 수 있는 devtools
2-4. 비동기 상태 관리
=> API 호출 결과를 상태로 관리할 수 있으며, 상태 업데이트와 비동기 작업을 결합할 수 있습니다.
2-5. 작고 가벼운 라이브러리
2-6. React와 잘 통합됨


3. zustand 단점
3-1. 대규모 애플리케이션에서의 관리 어려움
3-2. 상태의 복잡도 관리
=> 상태 객체가 깊고 중첩된 구조일 경우, 상태 업데이트가 복잡해질 수 있습니다.
3-3. 중복 상태 및 논리 처리
