1. Mocking
=> 테스트에서 의존성을 가짜로 대체하여 테스트 대상을 독립적으로 검증할 수 있도록 하는 기법입니다.
=> 코드가 의존하는 외부 요소(네트워크 요청, 데이터베이스, 타 모듈 등)를 실제로 실행하지 않고, 
 테스트 환경에서 대신 작동하는 "가짜 객체"나 "가짜 함수"를 사용합니다.


2. Mocking이 중요한 이유
2-1. 독립성 보장: 외부 요소와 분리하여 테스트 대상만 검증할 수 있습니다.
2-2. 성능 향상: 네트워크 요청이나 데이터베이스 작업처럼 시간이 오래 걸리는 작업을 Mock으로 
 대체해 테스트 속도를 빠르게 합니다.
2-3. 예측 가능성: 외부 환경(네트워크 지연, API 응답 변화 등)의 영향을 받지 않으므로 테스트 결과가 항상 일정합니다.
2-4. 에러 시뮬레이션: 외부 서비스의 실패 시나리오를 재현하여 에러 처리 로직을 테스트할 수 있습니다.
