1. TDD(테스트 주도 개발)
=> TDD는 테스트를 먼저 작성하고, 그 테스트를 통과시키기 위해 코드를 작성하는 소프트웨어 개발 방법론입니다.

1-1. 흐름
1. 테스트 작성: 요구사항에 맞는 실패하는 테스트를 먼저 작성합니다.
2. 코드 작성: 테스트를 통과시키기 위한 최소한의 코드를 작성합니다.
3. 팩토링: 코드를 개선하면서도 테스트가 계속 통과하도록 유지합니다.


2. BDD(행동 주도 개발)
=> BDD는 TDD에서 발전된 방법론으로, 비즈니스 요구사항과 사용자의 행동에 초점을 맞춘 테스트 기반 개발입니다.

2-1. 핵심 특징
1. 언어: 테스트를 작성할 때 사람이 이해할 수 있는 자연어 스타일(예: Given-When-Then)을 사용합니다.
2. 협업 중심: 개발자, QA, 비즈니스 담당자 간의 협업을 강조합니다.


3. 차이점
3-1. TDD는 개발자 관점, BDD는 비지니스 요구사항 관점
3-2. TDD는 기술적인 테스트를 기반으로 코드 작성 순서를 정의합니다.
3-3. BDD는 사용자의 행동과 요구사항 중심으로 시스템 설계를 주도합니다.
