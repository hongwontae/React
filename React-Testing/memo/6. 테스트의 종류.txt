1. Unit Test
=> 가장 작은 단위의 코드(컴포넌트, 함수, 메서드 등)를 독립적으로 테스트합니다.

1-1. 목적
=> 컴포넌트나 함수가 독립적으로 올바르게 동작하는지 확인합니다.
=> 다른 모듈이나 의존성의 영향을 배제하고 해당 코드만 테스트입니다.


2. Integration Test (통합 테스트)
=> 여러 컴포넌트나 모듈이 함께 작동할 때 상호작용이 올바르게 이루어지는지 확인합니다.

2-1. 목적
=> 여러 유닛이 결합될 때 발생할 수 있는 문제를 식별합니다.
=> UI와 비즈니스 로직 간의 연결이 올바른지 확인합니다.


3. Functional Test (기능 테스트)
=> 애플리케이션의 특정 기능이 예상대로 작동하는지 확인합니다.

3-1. 목적
=> 사용자가 기대하는 대로 특정 기능이 동작하는지 확인합니다.
=> UI 이벤트와 비즈니스 로직의 결합된 흐름을 검증합니다.


4. Acceptance Test / End-to-End (E2E) Test (인수 테스트 / 끝-끝 테스트)
=> 사용자가 애플리케이션을 실제로 사용하는 환경에서 전체적인 사용자 시나리오를 검증합니다.

4-1. 목적
=> 애플리케이션의 전체 흐름이 문제없이 작동하는지 확인합니다.
=> 프론트엔드, 백엔드, 데이터베이스 등 전체 스택을 포함한 통합 테스트입니다.
