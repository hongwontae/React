1. Assertion(단언)
=> 테스트의 통과 여부를 결정하는 부분입니다.

2. expect 메서드
=> Jest, Vitest의 전역 메서드입니다.

2-1. expect()
=> 인자를 조사하여 기대를 충족하는지 확인합니다.
=> expect()를 호출한 이후 메서드 체이닝으로 매처를 사용합니다.

2-2. Matcher
=> 단언 유형을 나타냅니다.
=> toBeInTheDocument() : 요소가 문서에 있는지 없는지를 나타내는 매처입니다.
=> toBe() : 인자의 값이 매처의 인자 값과 같기를 나타내는 매처입니다.
=> toHaveLength() : 인자의 길이가 매처의 인자의 길이와 같기를 원하는 매처입니다.
