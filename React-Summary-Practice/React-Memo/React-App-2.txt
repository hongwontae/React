4. useEffect();
4-1. useEffect의 콜백 함수의 본문의 실행은 해당 컴포넌트의 모든 렌더링이 끝난 이후이다.
4-2. useEffect의 두 번쨰 인자인 의존성 배열의 변화 여부에 따라 Effect 함수의 재실행 여부를 판단한다.
=> 컴포넌트 렌더링 완료 => useEffect의 콜백 함수 실행 => 컴포넌트가 다시 렌더링이 된다. => 이 떄 useEffect의 재실행 여부는
의존성 배열의 값의 변화이다. 의존성 배열이 변경되면 useEffect을 다시 실행시킨다.
4-3. 의존성이 배열의 값이 없다면 useEffect 재실행이 될 일은 절대 없다.
